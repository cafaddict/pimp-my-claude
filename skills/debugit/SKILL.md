## 디버깅 프로토콜

반드시 아래 순서를 따라 진행하라. 단계를 건너뛰지 마라.

### 1. 증상 정리
- 에러 메시지 / 스택 트레이스 / 비정상 동작을 정확히 기록
- 재현 조건 확인 (항상? 간헐적? 특정 입력?)
- 관련 파일과 함수 범위 특정

### 2. 가설 수립 (최소 3개)
- 가능한 원인을 3가지 이상 나열
- 각 가설의 근거를 한 줄로 설명
- 가장 가능성 높은 순서로 정렬

### 3. 가설 검증
- 가설마다 검증 방법을 정하고 실행
- 로그 추가, 조건 변경, 단위 테스트 등
- 검증 결과를 기록: ✅ 확인 / ❌ 반증 / ❓ 불확실

### 4. 근본 원인 특정
- 검증된 원인을 명확히 한 문장으로 서술
- "왜 이런 버그가 생겼는가"까지 설명 (단순 "어디"가 아닌 "왜")

### 5. 수정
- 최소 변경 원칙으로 수정
- 수정 전후 diff를 보여주고 변경 이유 설명
- 관련 없는 코드 리팩토링 금지

### 6. 검증
- 기존 테스트 실행하여 회귀 없음 확인
- 원래 문제가 해결되었는지 확인
- 엣지 케이스 테스트 추가 (필요시)
