## 프롬프트 변환기

사용자가 입력한 자연어 요청을 분석하여 아래 구조로 변환하라.

### Task/Context/Requirements/Output 구조

```
Task: [무엇을 해야 하는지 — 한 문장]

Context: [왜 필요한지, 배경 정보, 관련 파일/모듈]

Requirements:
- [구체적 조건 1]
- [구체적 조건 2]
- [제약사항]

Output: [코드? 설명? JSON? diff?]
```

### 모드 판별

요청의 성격에 따라 적합한 모드를 제안하라:

| 모드 | 언제 | 특징 |
|------|------|------|
| **Build** | 새 코드 작성, 기능 구현 | 코드 작성에만 집중 |
| **Learn** | 개념 이해, 설명 요청 | 왜 이런 구조인지 설명 |
| **Critique** | 코드 검토, 개선점 찾기 | /review 스킬 사용 권장 |
| **Debug** | 에러 수정, 문제 해결 | /debug 스킬 사용 권장 |

### 스코프 판단

- 3개 이상 파일 수정 예상 → **Plan Mode 먼저** 권장
- 다단계 작업 → **/taskloop 사용** 권장
- 스코프가 넓으면 → **분할 제안** (각각 독립 세션)

### 출력

1. 변환된 구조화 프롬프트
2. 권장 모드
3. Plan Mode / taskloop / 분할 필요 여부
4. 불명확한 점이 있으면 질문
