## 병렬 리서치 팀

여러 researcher를 병렬로 실행하여 다양한 관점에서 조사하고, 결과를 종합한다.
Subagent 패턴 — 독립 조사이므로 팀원 간 통신 불필요.

### 프로세스

#### 1. 리서치 질문 분해

$ARGUMENTS를 분석하여 독립적인 조사 관점으로 분해하라.

일반적인 분해 패턴:
- **기술 선택**: 각 후보를 별도 리서처가 조사
- **문제 해결**: 공식 문서 / 커뮤니티 사례 / 코드베이스 분석 각각
- **비교 분석**: 장점 담당 / 단점 담당 / 대안 탐색 각각
- **깊은 조사**: 이론/학술 / 실전 사례 / 현재 코드베이스 적용 가능성 각각

보통 3-4명이 적절. 5명 이상은 비용 대비 효과 낮음.

#### 2. 병렬 리서치 실행

각 관점에 대해 `researcher` 에이전트를 **병렬로** 실행.

```
관점별로 Agent 도구를 병렬 호출:
- subagent_type: researcher (또는 general-purpose)
- prompt: "[관점]에서 [주제]를 조사하라. 출처 링크 필수."
```

각 리서처에게 명확한 범위를 지정하여 중복 조사 방지.

#### 3. 결과 종합

모든 리서처 완료 후:
- 공통 발견 정리
- 충돌/모순 식별 및 판단
- 관련성 순위 매기기
- 최종 권고안 작성

#### 4. vault 저장

종합 결과를 vault에 저장 (note 스킬 사용):
- 기술 선택 → `decisions/`
- 일반 리서치 → `resources/`

#### 5. 출력

```
## 리서치 종합: [주제]

### 핵심 발견
- [공통으로 확인된 사항]

### 관점별 요약
| 관점 | 핵심 발견 | 출처 |
|------|----------|------|

### 충돌/모순
- [리서처 간 의견이 다른 부분과 판단]

### 권고
[근거 기반 최종 권고]

### 출처
[전체 링크 목록]
```

### 주의
- 리서처에게 Write/Edit 권한 주지 마라 (프로젝트 파일 수정 방지)
- 같은 소스를 여러 리서처에 중복 할당하지 마라
- 종합 없이 개별 결과만 나열하지 마라
