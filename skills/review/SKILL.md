## 리뷰 컨텍스트

- 변경사항: !`gh pr diff $0 2>/dev/null || git diff`
- PR 설명: !`gh pr view $0 2>/dev/null || echo "PR 없음 — git diff 기반 리뷰"`

## 리뷰 프로토콜

### 1. 변경 범위 파악
- 변경된 파일 목록과 변경 규모 정리
- PR 설명/커밋 메시지에서 의도 파악

### 2. 정확성 검토
- 로직이 의도대로 동작하는가?
- 엣지 케이스 처리가 되어있는가?
- 에러 핸들링이 적절한가?

### 3. 보안 검토
- 인젝션 (SQL, command, XSS) 가능성
- 인증/인가 우회 가능성
- 민감 데이터 노출 (로그, 에러 메시지)

### 4. 성능 검토
- 불필요한 연산, N+1 쿼리, 메모리 누수
- 알고리즘 복잡도가 적절한가?
- 대량 데이터에서의 동작

### 5. 테스트 커버리지
- 변경된 로직에 대한 테스트가 있는가?
- 엣지 케이스 테스트가 포함되어 있는가?
- 테스트가 누락되었다면 어떤 테스트가 필요한가?

## 출력 형식

> 수정 제안

**[Suggestion]** `파일:라인` — 설명
> 수정 제안
