## Boris 스타일 태스크 루프

$ARGUMENTS가 비어있으면 사용자에게 태스크명을 물어라.

### 디렉토리 구조

```
.claude/tasks/
├── $ARGUMENTS/
│   ├── todo.md       ← 이 태스크의 계획/진행 추적
│   └── lessons.md    ← 이 태스크에서 얻은 교훈
└── lessons.md        ← 프로젝트 전체 공유 교훈
```

### 프로세스

#### 1. 초기화
- `.claude/tasks/$ARGUMENTS/` 디렉토리 생성 (없으면)
- 기존 `todo.md`가 있으면 이어서 진행
- `.claude/tasks/lessons.md`가 없으면 생성

#### 2. 계획 작성 (todo.md)
아래 형식으로 계획을 작성하라:

```markdown
# 태스크: $ARGUMENTS

## 목표
[한 문장으로 이 태스크의 목표]

## 계획
- [ ] 단계 1: [구체적 작업]
- [ ] 단계 2: [구체적 작업]
- [ ] 단계 3: [구체적 작업]

## 상태
- 시작: [날짜]
- 현재: 계획 작성 완료, 승인 대기

## 메모
[진행 중 발견한 사항]
```

#### 3. 승인 대기
**반드시 사용자의 승인을 받은 후에만 구현을 시작하라.**
"이 계획으로 진행할까요?" 라고 물어라.

#### 4. 실행
- 각 단계 완료 시 todo.md의 체크박스를 `[x]`로 업데이트
- 상태 섹션도 업데이트
- 예상치 못한 문제 발생 시 메모 섹션에 기록

#### 5. 교훈 기록
실수나 의미있는 발견이 있으면 두 곳에 기록:
- `.claude/tasks/$ARGUMENTS/lessons.md` — 이 태스크 한정
- `.claude/tasks/lessons.md` — 프로젝트 전체에 적용되는 교훈

교훈 형식:
```markdown
### [날짜] [한 줄 요약]
- **상황**: 무엇을 하려 했는가
- **문제**: 무엇이 잘못되었는가
- **교훈**: 다음에 어떻게 해야 하는가
```

#### 6. 완료
- todo.md 모든 체크박스 완료
- 상태를 "완료"로 업데이트
- 최종 결과 요약 추가
